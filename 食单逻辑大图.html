图1 · 信任分层架构：交易是地基，收藏是装饰
食单的信任不是单一来源，而是分层叠加 —— 交易证据做承重墙，收藏意图做扩展，分享行为做放大器
分享行为 社交背书
用户主动分享 = 用个人信用为食单背书，信任放大到社交圈
收藏行为 想去
意图信号，"我想去"≠"我去过"
可扩展个性，但不可做信任基座
交易记录 已验证
不可伪造的消费事实
美团独有资产，信任的承重墙
地基层 只有交易验证的店才能撑起食单的信任底线
没有交易的食单 = 收藏夹，不是食单
装饰层 收藏的店标注"想去"，丰富个性表达
但不承担信任责任
放大层 分享行为将个人信任扩散到社交网络

用户决策的三个支撑工具，回答完全不同的问题，不可互相替代

回答同 / 异（身份投射）

消费证据的信
核心洞察：推荐流是工具（utility），解决"现在吃什么"；食单是身份（identity），回答"我是什么样的人"。推荐流用完即走，食单可拥有可分享。差异不在于生成方式（都是算法），而在于用户能否对其施加意志——能编辑、能策展、能拥有，才是食单。
图3 · 食单 vs 推荐流：差异根源拆解
同样是算法生成，为什么食单不是"推荐流换皮"？差异在五个维度上同时成立
维度 推荐流 个性化食单
可拥有性 即时流，每次打开都不同，无法"拥有" 有限稳定集合，有边界，"这是我的"
用户意志 不可编辑，算法全权决定 系统生成草稿 → 用户编辑定稿（增删/排序/备注）
心理机制 不触发禀赋效应（非拥有物） 触发禀赋效应 + IKEA效应（投入劳动→拥有感倍增）
使用目的 问题解决："现在吃什么" → 用完即走 身份投射："我是什么样的人" → 反复回看
社交属性 不可分享（无法分享一个流） 可分享（有限集合 + 消费证据 = 社交货币）
关键结论：如果个性化食单只是系统生成、用户不能编辑，那它就是推荐流的另一种展示——你的直觉完全正确。真正产生差异的是用户编辑环节：系统生成是"草稿"，用户编辑是"定稿"。用户删掉两家店、加一条备注、调一下排序——经过这个动作，"算法输出"变成"我的食单"。没有编辑权 = 没有拥有感 = 没有差异。

第一层

零成本信任获客通道

第二层
信号
第四种商家质量信号
"被N个用户的食单收录" = 群体消费背书。
这是区别于评价（个体/噪音）、榜单（权威/头部）、销量（交易/不解释why）的第四种信号。 它衡量的是"这家店值不值得被记住"——不是好不好吃，而是有没有复访价值。
反哺路径：食单收录数据 → 榜单算法（长尾质量发现）→ 推荐模型（冷启动供给）→ 商业化决策（谁值得被推）
vs 评价/榜单 评价：个体主观，易刷
榜单：周期性，覆盖窄
食单信号：群体行为，难伪造

关系

图5 · 冷启动实战路径：三个假设，逐层验证
食单冷启动的独特优势：不需要从零生产内容（消费数据已有），真正要验证的是三个递进假设——拥有感、分享欲、转化力
核心洞察：大多数UGC产品冷启动是"没内容→没用户→没内容"的死循环。食单不存在这个问题——内容=消费数据，已经存在。真正的冷启动挑战是三个递进假设：H1 用户会编辑食单吗？（拥有感）→ H2 用户会分享食单吗？（社交货币）→ H3 接收方会因食单进店吗？（信任转化）。每个假设必须独立验证，前一个不成立，后面的没有意义。
PHASE 0
基础设施
2-3周 · 上线前
要建什么
消费数据聚合引擎：不是全量流水，而是按场景智能分组（如"你常去的粉店""周末聚餐精选""公司附近午餐"）
食单草稿生成算法：从交易记录中筛选+分组+自动命名
编辑工具：增删/排序/备注/封面图/食单命名
分享链路：生成分享卡片→接收页→转化追踪埋点
关键设计原则
草稿不能是全量交易记录堆砌——那叫流水账不叫食单
分组逻辑要让用户看到时觉得"哦这确实是我"
编辑工具要轻：一步就能删/加店，两步就能写备注
没有编辑工具 = 食单=推荐流 = 项目失败
PHASE 1
拥有感验证
2城市 · 3-4周
核心假设 H1
用户会编辑系统生成的食单草稿，产生"这是我的"拥有感
用户策略：高频交易用户优先（top 20%）
触发机制：消费后push"你的食单已更新" + 主动入口"我的食单"
城市选择：长沙/成都（年轻+高频+社交活跃）
指标 & 失败信号
食单打开率 编辑率（至少增删/排序/备注一项） 编辑深度（平均编辑动作数） 二次打开率
失败信号：打开率高但编辑率<5% → 用户觉得"看看就行"，没有拥有感。此时分享和转化都不用验证了——地基没立住。
PHASE 2
分享验证
同2城市 · 3周
核心假设 H2
用户愿意把食单作为社交货币分享出去
前提：Phase 1编辑率达标
分享场景设计（不在通用入口放按钮，场景化触发）：
· 朋友问"附近有什么好吃的" → 推荐分享食单
· 周末/节假日 → "把你的美食地图分享给朋友"
· 朋友来访 → "为朋友生成一份你的城市食单"
分享物：卡片+"已验证"标识+食单个性摘要
指标 & 失败信号
分享率（已编辑食单中分享占比） 分享后打开率 分享场景触发率
失败信号：分享率<2% → 食单不具有社交货币属性。调整方向：降低分享门槛（一键分享）或增强食单的"可炫耀性"（如生成"我的年度美食报告"式卡片）。
PHASE 3
转化验证
同2城市 · 3周
核心假设 H3
接收方会因为食单（而非其他原因）进店消费
前提：Phase 2分享率达标
转化路径优化：食单分享页→店铺详情→下单
信任展示：突出"X人已验证""基于真实消费"
对照实验：A/B Test — 食单进店vs推荐流进店，对比转化率和客单价
指标 & 失败信号
食单页→店铺详情转化率 食单→下单转化率 vs 推荐流转化率差异 长尾店食单转化vs头部店
失败信号：转化率≤推荐流 → 食单信任没有产生决策价值。核心问题可能在：食单展示不够吸引/接收方不认识分享者/食单中的店距离太远。
PHASE 4
激励验证
同2城市 · 2周
核心假设 H4
优质食单激励能放大飞轮，而非制造垃圾内容
前提：H1-H3全部验证
激励设计：不是人人激励，是转化率top 20%的食单获得券/流量
激励对象：高编辑深度+高转化率的食单创作者
防垃圾：激励基于转化结果而非创建行为——只有别人真的因为你的食单进了店，你才拿奖励
指标 & 失败信号
激励后分享率提升幅度 激励食单二次传播率 低质食单占比（防垃圾）
失败信号：低质食单占比>30% → 激励被刷。调整：提高激励门槛（需3次以上转化才激励）或改为平台邀请制（只邀请高质食单创作者）。
扩量决策门
绿灯 · 扩量
H1-H3全部验证 + H4激励有效 → 扩展至5-8城市，增加品类覆盖
黄灯 · 修复
部分假设验证 → 针对失败假设做产品修复（如编辑门槛/分享场景/转化路径），重验一轮
红灯 · 回退
H1失败（用户不编辑）→ 根本性回退：用户是否真的需要"食单"这个形态？回到需求验证
